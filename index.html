REAR: Retrieve, Expand and Refine for Effective Multitable Retrieval

About

Answering natural language queries over relational data often requires retrieving and reasoning over multiple tables, yet most retrievers optimize only for query–table relevance and ignore table–table compatibility. We introduce REAR (Retrieve, Expand and Refine), a three-stage, LLM-free framework that separates semantic relevance from structural joinability for efficient, high-fidelity multi-table retrieval. REAR (i) retrieves query-aligned tables, (ii) expands these with structurally joinable tables via fast, precomputed column-embedding comparisons, and (iii) refines them by pruning noisy or weakly related candidates.

TL;DR: REAR is a three-stage retrieval framework that jointly optimizes query–table relevance and table–table joinability without online LLM calls, achieving performance competitive with state-of-the-art LLM-augmented systems while maintaining significantly lower latency and cost.

Framework Overview

REAR comprises three stages that cleanly separate different aspects of multi-table retrieval:

  • Stage 1 - Retrieve: Select top-k relevant tables using standard dense, sparse, or hybrid retrievers to ensure query-table relevance.
  • Stage 2 - Expand: Augment with structurally joinable tables using FAISS column search and cross-encoder reranking, expanding from k to k'+∆k' candidates.
  • Stage 3 - Refine: Score candidates by both query relevance and table joinability, reranking to final top-k tables.

Key Features

  • LLM-Free Design: Eliminates online LLM calls, achieving 92.24% token reduction compared to ARM
  • Retriever-Agnostic: Works as a plug-and-play layer over any standard retriever (dense, sparse, or hybrid)
  • Joint Optimization: Explicitly models both query–table relevance and table–table joinability
  • Efficient Expansion: Uses precomputed column embeddings with FAISS for sublinear-time joinability detection
  • Precision-Focused Refinement: Prunes noisy candidates while maintaining high recall

Methodology

1. Retrieval Stage

The retrieval stage identifies base tables semantically relevant to the query using standard retrievers:

  • Sparse retrievers: TF-IDF based scoring (BM25, SPLADE)
  • Dense retrievers: Cosine similarity between learned embeddings (UAE, GTE, e5-mistral, BGE)
  • Hybrid retrievers: Weighted combination of sparse and dense scores

2. Expansion Stage

The expansion stage adds tables that are structurally joinable with the base set through:

  • Detection: Identify joinable tables by measuring column embedding similarity using approximate nearest neighbor search
  • Reranking: Filter semantically distant tables using cross-encoder reranking to ensure both joinability and query relevance

3. Refinement Stage

The refinement stage jointly scores candidates using:

  • Query-table relevance: Cross-encoder similarity between query and table
  • Table-table joinability: Attention-weighted score based on column-level similarities
